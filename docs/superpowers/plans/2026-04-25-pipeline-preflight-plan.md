# Sub-project 0 — Pipeline Pre-flight Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prove the Windows-only Expo dev pipeline end-to-end. Default `create-expo-app` starter runs in Expo Go on the user's iPhone, updating on hot reload.

**Architecture:** No application code. Install toolchain, scaffold Expo into the existing `pulse/` repo, configure bundle identifier, verify the iPhone runtime connects to the dev server.

**Tech Stack:** Node.js LTS, Expo CLI, Expo SDK 54 (TypeScript, New Architecture), Expo Go on iPhone.

> **Scope change 2026-04-25:** Android emulator path skipped — Pulse is iPhone-first (HealthKit forces this), so Android Studio install + Pixel 7 emulator + Task 9 (Android run) are dropped. Tasks 1–3 are also moot: Node was already installed (v22.14), and Tasks 2–3 covered Android Studio. SDK landed as **54** (latest as of today), not 55 as the plan originally guessed.

## Task 2: Install Android Studio

**Files:** none

- [ ] **Step 1: Check whether Android Studio is already installed**

Open the Start menu and search for "Android Studio". If it opens, **skip to Task 3**.

- [ ] **Step 2: Download the installer**

Browse to https://developer.android.com/studio → click "Download Android Studio" → accept terms → download the Windows `.exe`.

- [ ] **Step 3: Run the installer**

Accept all defaults. The installer will download and unpack the Android SDK on first launch — this takes 10–20 minutes depending on connection.

- [ ] **Step 4: First-launch setup wizard**

Launch Android Studio. When the setup wizard appears:
- Choose **Standard** install type
- Accept all licenses
- Let it download SDK + Platform Tools + Build Tools (~3–5 GB)

- [ ] **Step 5: Verify SDK installed**

In Android Studio: **More Actions** (or **Configure**, depending on splash version) → **SDK Manager**. Confirm Android API 34 (or 35 if available, prefer 34 for stability) shows "Installed" under the **SDK Platforms** tab. If not, check it and click Apply.

- [ ] **Step 6: Add Android SDK env vars**

Find the SDK path shown at the top of SDK Manager (typically `C:\Users\<you>\AppData\Local\Android\Sdk`).

Open Windows Settings → System → About → Advanced system settings → Environment Variables → User variables → New:

| Variable | Value |
|---|---|
| `ANDROID_HOME` | `C:\Users\<you>\AppData\Local\Android\Sdk` |
| `ANDROID_SDK_ROOT` | (same as above) |

Edit `Path` and add:
- `%ANDROID_HOME%\platform-tools`
- `%ANDROID_HOME%\emulator`

Click OK on every dialog. **Close all terminals.**

- [ ] **Step 7: Verify adb is on PATH**

Open a fresh terminal. Run: `adb --version`
Expected: `Android Debug Bridge version 1.0.x …`

## Task 3: Create and boot a Pixel 7 / API 34 emulator

**Files:** none

- [ ] **Step 1: Open Device Manager**

In Android Studio: **More Actions** → **Virtual Device Manager**.

- [ ] **Step 2: Create a Pixel 7 device**

- [ ] **Step 3: Boot the AVD**

In Device Manager, click the ▶ Play icon next to `Pixel_7_API_34`. The emulator window opens and boots to the Android lock/home screen. First boot can take 2–3 minutes.

In a terminal: `adb devices`
Expected:
```
List of devices attached

```

If shown as `offline`, wait 30 seconds and re-run.

## Task 4: Scaffold the Expo app into the existing repo (collision-safe)

**Files:**
- Create: dozens of files in `pulse/` from `create-expo-app` template
- Preserve: `README.md`, `STACK.md`, `CLAUDE.md`, `design_handoff/`, `docs/`

`create-expo-app` won't run in a non-empty directory. We'll scaffold to a temp sibling, then merge the files in, preserving the ones we already have.

Run: `pwd && ls`
Expected: working directory ends in `pulse`, listing shows `README.md`, `STACK.md`, `CLAUDE.md`, `design_handoff/`, `docs/`.

- [ ] **Step 2: Scaffold into a sibling temp dir**

Run: `cd .. && npx create-expo-app@latest pulse-scaffold --no-install`

Expected output ends with something like:
```
✅ Your project is ready!
```
A new `pulse-scaffold/` dir appears alongside `pulse/`.

- [ ] **Step 3: Inspect what the scaffold produced**

Run: `ls pulse-scaffold/`
Expected files include: `app/`, `assets/`, `components/`, `constants/`, `hooks/`, `scripts/`, `app.json`, `package.json`, `tsconfig.json`, `.gitignore`, `README.md`, `expo-env.d.ts`.

- [ ] **Step 4: Move scaffold contents into pulse/, preserving our files**

In Git Bash (not PowerShell — globbing differs):
```bash
cd pulse-scaffold
# move everything except README.md (we keep ours)
for item in $(ls -A); do
  if [ "$item" != "README.md" ]; then
    mv "$item" ../pulse/
  fi
done
cd ..
rm -rf pulse-scaffold
cd pulse
ls
```

Expected `ls` after: original files PLUS `app/`, `assets/`, `components/`, `constants/`, `hooks/`, `scripts/`, `app.json`, `package.json`, `tsconfig.json`, `expo-env.d.ts`. The `.gitignore` from the scaffold replaced the old one (or was added if there wasn't one).

- [ ] **Step 5: Install dependencies**

Run: `npm install`
Expected: completes without errors. Will take 1–3 minutes.

- [ ] **Step 6: Sanity-check Expo CLI works in the project**

Run: `npx expo --version`
Expected: a version like `0.x.x` (Expo CLI version, not SDK).

Run: `npx expo doctor`
Expected: a series of checks. All should be ✅ or warnings (warnings about Expo Router setup are fine; we're not testing routes in pre-flight).

- [ ] **Step 7: Commit the scaffold**

```bash
git add -A
git commit -m "chore: scaffold Expo SDK 55 app (sub-project 0)"
```

## Task 5: Configure bundle identifier, display name, and New Architecture

**Files:**
- Modify: `app.json`

- [ ] **Step 1: Open app.json**

Read the current contents. The Expo template puts the config under an `"expo"` key. Find the existing `"name"`, `"slug"`, and look for `"ios"` and `"android"` blocks.

- [ ] **Step 2: Set name, slug, bundle identifier, Android package, and New Architecture**

Edit `app.json` so the `"expo"` object includes (merging with what's already there — don't delete existing keys like `"icon"` or `"splash"`):

```json
{
  "expo": {
    "name": "Pulse",
    "slug": "pulse",
    "newArchEnabled": true,
    "ios": {
      "bundleIdentifier": "com.kael.pulse",
      "supportsTablet": true
    },
    "android": {
      "package": "com.kael.pulse",
      "adaptiveIcon": {
        "foregroundImage": "./assets/images/adaptive-icon.png",
        "backgroundColor": "#ffffff"
      }
    }
  }
}
```

(Keep the rest of the template's keys: `version`, `orientation`, `icon`, `userInterfaceStyle`, `splash`, `assetBundlePatterns`, `web`, `plugins`, etc.)

- [ ] **Step 3: Verify with expo doctor**

Run: `npx expo doctor`
Expected: still passing. If a new warning appears about the bundle identifier, read it — it should be informational, not an error.

- [ ] **Step 4: Commit**

```bash
git add app.json
git commit -m "chore: set Pulse name and com.kael.pulse bundle id"
```

## Task 8: Allow port 8081 through Windows Firewall

**Files:** none

Metro (Expo's bundler) listens on port 8081. Windows Defender Firewall typically prompts on first launch, but we'll pre-authorize to avoid the prompt blocking the iPhone connection.

- [ ] **Step 1: Open Windows Defender Firewall with Advanced Security**

Start menu → search "Windows Defender Firewall with Advanced Security" → open.

- [ ] **Step 2: Create an inbound rule for port 8081**

Inbound Rules → New Rule → Port → TCP → Specific local ports: `8081` → Allow the connection → All profiles (Domain, Private, Public — Public is required for many home networks) → Name: `Expo Metro Bundler`. Finish.

- [ ] **Step 3: Verify the rule appears**

Inbound Rules list → confirm `Expo Metro Bundler` row exists with Action = Allow.
